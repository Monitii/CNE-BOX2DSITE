Welcome to CNE-Box2D

This is a WIP port of Box2D physics engine to Codename Engine (FNF). Add Physics to anything, make BF bouncy, Make a huge wrecking ball that kills everyone!!! (Almost) anything is possible! And the docs are here to explain everything possible.

Coordinate Conversion

pxToWorldX(pixels)
pxToWorldY(pixels)
Convert pixel coordinates to Box2D world units (meters). Box2D uses meters as its base unit, so you'll need to convert your pixel positions before passing them to physics functions.
worldToPxX(meters)
worldToPxY(meters)
Convert Box2D world units (meters) back to pixel coordinates. Use these to position your visual sprites based on physics body positions.

Initialization

box2d_init()
Loads all Box2D functions from the NDLL (Native Dynamic Link Library). Call this first! Returns true if successful, false otherwise.

World Management

box2d_createWorld(gravityX, gravityY)
Creates a physics world with specified gravity. Returns a world ID that you'll use for all other operations. Typical gravity: gravityX = 0, gravityY = 9.8 (or higher for more dramatic effect).
box2d_step(worldId, timestep, velocityIterations, positionIterations)
Advances the physics simulation by one timestep. Call this every frame to update the physics world. Recommended values: timestep = 1/60, velocityIterations = 8, positionIterations = 3.

Body Creation

box2d_createBody(worldId, x, y, type)
Creates a physics body at the specified position (in pixels). Returns a body ID.

Body types:
• 0 = static - Never moves (floors, walls)
• 1 = kinematic - Moves but unaffected by forces (moving platforms)
• 2 = dynamic - Full physics simulation (players, objects)
box2d_addBox(bodyId, width, height, offsetX, offsetY)
Adds a rectangular collision shape (fixture) to a body. Size and offset are in pixels.
box2d_addCircle(bodyId, radius, offsetX, offsetY)
Adds a circular collision shape to a body. Radius and offset are in pixels.

Body Getters

box2d_getPos(bodyId)
Gets the body's position as an array [x, y] in pixels.
box2d_getPosObj(bodyId)
Gets the body's position as an object {x: value, y: value} in pixels.
box2d_getAngle(bodyId)
Gets the body's rotation angle in radians. Convert to degrees with angle * 180 / Math.PI.
box2d_getFixtures(bodyId)
Gets a list of all collision shapes (fixtures) attached to this body.
box2d_getAngVel(bodyId)
Gets the body's angular velocity (how fast it's rotating) in radians per second.

Body Setters

box2d_setBodyTransform(bodyId, x, y, angle)
Directly sets the body's position (in pixels) and rotation (in radians). Use sparingly - prefer forces/impulses for realistic physics.

Forces & Impulses

box2d_impulse(bodyId, forceX, forceY)
Applies an instant linear force to the body's center (like a kick or explosion). Great for jumps, knockback, and instant velocity changes.
box2d_impulseVec(bodyId, forceVector)
Same as box2d_impulse but takes a vector/object parameter: {x: forceX, y: forceY}.
box2d_torque(bodyId, torque)
Applies a rotational force to make the body spin. Positive values spin clockwise, negative values spin counter-clockwise.
box2d_setAngVel(bodyId, velocity)
Directly sets the rotation speed in radians per second. Use for controlled spinning.

Physical Properties (Fixture)

box2d_setRestitution(fixtureId, value)
Sets the bounciness of a collision shape.
• 0.0 = No bounce (soft landing)
• 1.0 = Perfect bounce (basketball)
• >1.0 = Gains energy (super bouncy!)
box2d_setFriction(fixtureId, value)
Sets the surface friction.
• 0.0 = Ice (no friction)
• 0.3 = Default (normal surface)
• 1.0+ = Sticky/rough surface
box2d_setDensity(fixtureId, value)
Sets the density which affects mass and weight. Higher density = heavier object (harder to move, more momentum).

Physical Properties (Body)

box2d_setLinearDamping(bodyId, value)
Sets air resistance for movement.
• 0.0 = No air resistance
• Higher values = More drag (slower movement over time)
box2d_setAngularDamping(bodyId, value)
Sets air resistance for rotation. Higher values make spinning slow down faster.
box2d_setGravityScale(bodyId, value)
Sets the gravity multiplier for this specific body.
• 1.0 = Normal gravity
• 0.0 = No gravity (floating)
• -1.0 = Reverse gravity (flies up!)
• 2.0 = Double gravity (heavy)

Joints

box2d_createRevoluteJointWorld(worldId, bodyId, anchorX, anchorY)
Pins a body to a fixed point in the world (like a hinge or pivot). The body can rotate around the anchor point but not move away from it. Perfect for pendulums, doors, or ragdoll limbs.
box2d_createRevoluteJointBetween(worldId, bodyA, bodyB, anchorX, anchorY)
Connects two bodies with a rotating joint at the specified anchor point. Both bodies can rotate freely around the connection point. Great for chains, rope bridges, or connected objects.
box2d_destroyJoint(worldId, jointId)
Removes a joint and disconnects the bodies. They'll become independent again.

Sync Sprite

box2d_syncSprite(bodyId, sprite, localX, localY, globalX, globalY)
Makes a visual sprite follow the position and rotation of a physics body with offsets to help.
Call this on draw.
Tip: Box2D often likes objects between 0.1 and 10 meters in size. for best results, stick to that range.
